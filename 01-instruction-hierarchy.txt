INSTRUCTION HIERARCHY

Your agents need to know where to find their instructions — and what to do when two files disagree.

You're going to create a chain of text files. Each level overrides the one below it:

  1. Your global AGENTS.md — lives in one repo, applies everywhere
  2. Your machine AGENTS.md — lives at ~/, describes this specific computer
  3. Each repo's AGENTS.md — project-specific rules
  4. Tool configs — CLAUDE.md, GEMINI.md, .cursor/rules/, copilot-instructions.md
  5. The tool's built-in defaults

If two files say different things, the higher-numbered file loses. Your global file always wins.

YOUR GLOBAL FILE

Pick one repo to hold your master instructions. Create an AGENTS.md there. This is where you put:

  How you want agents to behave (tone, verbosity, interaction style)
  Rules that apply to every project (commit conventions, formatting, etc.)
  Session start/end checklists
  Links to any coordination tools you use (optional)

Every agent on every machine reads this file first. Think of it as your constitution.

YOUR MACHINE FILE

Create ~/AGENTS.md on each machine you work on. This file points to your global file and describes the local environment:

  What OS and shell you're running
  Where your repos live
  What tools and runtimes are installed
  What services are running locally

This is a map, not a rulebook. Don't put rules here — those go in the global file.

YOUR PROJECT FILES

Each repo can have its own AGENTS.md for things that only matter in that project: build commands, test patterns, deploy steps, architecture notes.

TOOL CONFIGS

CLAUDE.md, GEMINI.md, .cursor/rules/*.mdc, .github/copilot-instructions.md — these handle quirks specific to one tool. They sit at the bottom of the chain and yield to everything above.

HOW AGENTS FIND THE CHAIN

Add an agents.txt file to each repo root. One line: the path to that repo's entry file. If the repo has its own AGENTS.md, agents.txt says "AGENTS.md". If not, it points to your global file. Any agent can follow this to find the full chain.
