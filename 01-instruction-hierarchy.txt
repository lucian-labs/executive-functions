INSTRUCTION HIERARCHY

Every agent resolves behavior through a strict precedence chain. Higher levels override lower:

  1. Global AGENTS.md — cross-machine canonical rules
  2. Machine AGENTS.md — per-machine entry point (~/AGENTS.md)
  3. Repo AGENTS.md — project-specific guidance
  4. Tool config — CLAUDE.md, GEMINI.md, copilot-instructions.md, .cursor/rules/
  5. Runtime defaults — whatever the tool ships with

If two sources conflict, the lower-numbered source wins. Always.

GLOBAL AGENTS.md

The root file. Lives in a designated canonical repo. Contains: working rules, interaction style, commit conventions, session protocols, coordination infrastructure links, product direction. Every agent on every machine reads this first. It is the constitution.

Typical sections: Mission, Global Working Rules, Interaction Style, Session Start Checklist, Session End Protocol, Coordination API, Product Direction.

MACHINE ENTRY POINT

~/AGENTS.md on each machine. Points to the global canonical file. Describes the local machine: OS, paths, installed tools, running services, hardware. Lists agent config locations. Defines bootstrap order.

This file is a map, not a rulebook. It contains no rules or conventions — those live in the global file.

REPO-LEVEL GUIDANCE

Each project can have its own AGENTS.md for project-specific rules: build commands, test patterns, architecture constraints, deploy procedures. Scoped to that repo only.

TOOL-SPECIFIC CONFIG

Bottom of the hierarchy. CLAUDE.md for Claude Code, GEMINI.md for Gemini, .github/copilot-instructions.md for Copilot, .cursor/rules/*.mdc for Cursor. Handle tool-specific quirks. Subordinate to everything above.

DISCOVERY CONTRACT

Each repo includes agents.txt at root — a single line pointing to the entry file for that repo. If the repo has its own AGENTS.md, agents.txt contains "AGENTS.md". If not, it points to the global file. This lets any agent find its way into the hierarchy from any starting point.
