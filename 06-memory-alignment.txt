MEMORY ALIGNMENT

Your operator model lives in files. Some AI tools also store their own memories between sessions — Claude has auto-memory, Cursor has a notepad, Codex has memory. This creates a drift risk: the tool's memory can diverge from your actual model. Memory alignment keeps your files as the single source of truth.

THE PROBLEM

You correct an agent. It updates its memory. A few sessions later, a different detail in memory goes stale. Over time the agent's behavior slowly diverges from your actual instructions — and you can't see it happening because memory is invisible to you.

THE FIX

Tell your agents (in the global AGENTS.md) that their memory is a cache, not a source of truth. Your files are the authority. If memory and files disagree, the files win.

WHAT TO PUT IN YOUR GLOBAL FILE

Add an alignment protocol. Something like:

  On session start:
    1. Read the instruction chain (AGENTS.md files)
    2. Compare your stored memories against those files
    3. If anything in memory contradicts a file, silently update the memory
    4. Don't ask about it. Don't mention it. Just fix it and move on.

  On session end:
    1. If you learned something that should be permanent, write it to the right file — not just to memory
    2. Memory should only hold: who I am, machine-specific details, pointers to where the real info lives

WHAT MEMORY SHOULD CONTAIN

  Your preferences — how you like to communicate, your expertise level, accessibility needs
  Machine-specific facts — paths, ports, hostnames that aren't in committed files
  Pointers — "the deploy process is in AGENTS.md under Deploy Rules"
  Recent corrections you've given that haven't been written into files yet

WHAT MEMORY SHOULD NOT CONTAIN

  Rules or conventions — those belong in your AGENTS.md
  Process definitions — those belong in your AGENTS.md or dedicated process files
  Task lists — those belong in your task system
  API docs — those belong at their source

THE RESULT

You maintain one set of files. Every agent self-corrects against them on startup. Memory stays in sync automatically. You never have to manually audit what an agent "remembers."

IF YOUR TOOL DOESN'T HAVE MEMORY

Some AI tools start completely fresh every session. For those, the instruction files are all you need. Memory alignment only matters for tools that carry state between sessions.
