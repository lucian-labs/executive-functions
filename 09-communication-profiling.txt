COMMUNICATION PROFILING

Before the system can work well for you, your agent needs to understand how you think, talk, and organize. Not through a questionnaire — through analysis of how you've actually been using AI tools.

WHAT HAPPENS DURING ONBOARDING

When you first set this up, your agent reads through whatever it can access about your existing interactions:

  Your prior conversations and chat history
  Any memory or notes the tool has stored about you
  Existing instruction files you've already written
  Your git commit messages and PR descriptions
  READMEs and docs you've authored
  Any writing samples or style guides you point it to

It's looking for patterns in how you behave — not what you talked about, but how you communicate.

WHAT IT'S LOOKING FOR

  How you give instructions: precise specs or broad intent? Do you iterate or expect first-pass accuracy?
  How you give feedback: do you explain why something's wrong or just redirect? What triggers frustration?
  How you think: lists, narratives, diagrams, compressed fragments? How much explanation do you want back?
  How you organize: by project, by urgency, by energy level, by domain?
  Your pace: do you batch instructions or stream them? Deep focus or frequent context-switches?
  Accessibility: ADHD, dyslexia, autism-spectrum, sensory preferences — anything affecting how info should be presented.
  Your tone: formal, casual, terse, playful, technical — or a blend that shifts by context.

THIS ISN'T A PERSONALITY TEST

It's calibration data. Your agent uses this to adjust everything: response length, structure, vocabulary, detail level, when to ask vs. when to act, how to format output.

Someone whose profile says "terse, compressed, ADHD, no preamble" gets fundamentally different agent behavior than someone whose profile says "detailed, exploratory, likes to think out loud."

WHERE IT GOES

The agent writes your communication profile into the interaction style section of your global AGENTS.md. Every agent on every platform reads it and calibrates the same way. No more re-training each tool separately.

If a tool has its own memory system, it can cache the profile for speed — but the file is always the authority (see 06-memory-alignment.txt).

IT GETS BETTER OVER TIME

The first profile is a draft. It refines itself:

  When you correct your agent's behavior, it updates the profile
  When patterns emerge across sessions (you consistently shorten long responses, you reformat lists into prose, you skip certain types of output), the agent notes them
  You can edit the profile directly any time — it's just a section in a text file

After a few sessions, corrections should become rare. The agent just works the way you work.

YOUR ORGANIZATION STYLE TOO

Beyond communication, the agent also picks up on how you organize:

  How you name things (files, branches, variables, projects)
  How you structure directories
  How you track tasks (formal system, scattered notes, or just your head)
  How you prioritize (deadlines, energy, interest, urgency)
  How you handle interruptions
  What "done" means to you

This shapes how the agent behaves during and after sessions — what it reports, what it tracks, what it leaves alone.

IF THERE'S NO HISTORY TO ANALYZE

No problem. The agent builds your profile from your first few interactions. It watches how you phrase your first instruction, how you respond to its first output, and whether you add context or expect discovery. Within 2-3 exchanges there's enough signal for an initial profile. It writes it immediately and refines from there.
